## Detailed Findings

### Weak/Default Credentials for Admin Account

**ID:** FINDING-001  
**Severity:** High (Analyst Estimate), Critical (CVSS 9.8)  
**OWASP Category:** [A07:2025 - Authentication Failures](https://owasp.org/Top10/2025/A07_2025-Authentication_Failures/)  
**CWE ID(s):** [CWE-521](https://cwe.mitre.org/data/definitions/521.html), [CWE-259](https://cwe.mitre.org/data/definitions/259.html)  
**CVSS Vector:** CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H  
**Severity Discrepancy Note:** Analyst assessed High; CVSS 3.1 calculates Critical (9.8) because the vulnerability allows full administrative access with default credentials, impacting all three CIA pillars.

#### Description
The application permits login using default credentials for the admin account, which poses a significant security risk. This vulnerability allows attackers to gain administrative access with ease, using commonly known credentials. Such access can lead to unauthorized data exposure, data manipulation, and potential service disruption.

#### Technical Detail
The login endpoint at `/rest/user/login` accepts the default credentials 'admin@juice-sh.op' with the password 'admin123', which grants admin-level access. This indicates a lack of proper credential management and password policy enforcement, leaving the system vulnerable to unauthorized access.

#### Evidence
A POST request to `http://localhost:3000//rest/user/login` using the credentials `admin@juice-sh.op:admin123` returned a valid session, confirming admin-level access.

#### Remediation
Immediately change the password for the admin account to a strong, unique password. Implement a policy to enforce strong passwords for all accounts. Consider disabling default accounts or requiring password changes upon first login. Additionally, implement strong password policies and enforce the use of unique, complex passwords for all accounts, especially administrative ones. Disable default credentials and require password changes upon first login.

- [CWE-521: Weak Password Requirements](https://cwe.mitre.org/data/definitions/521.html)
- [CWE-259: Use of Hard-coded Password](https://cwe.mitre.org/data/definitions/259.html)

### Site Served Over HTTP Only

**ID:** FINDING-002  
**Severity:** Medium (CVSS Score: 5.9)  
**OWASP Category:** [A04:2025 - Cryptographic Failures](https://owasp.org/Top10/2025/A04_2025-Cryptographic_Failures/)  
**CWE ID:** [CWE-319: Cleartext Transmission of Sensitive Information](https://cwe.mitre.org/data/definitions/319.html)  
**CVSS Vector:** CVSS:3.1/AV:N/AC:H/PR:N/UI:N/S:U/C:H/I:N/A:N  

#### Description
The application is currently accessible only over HTTP, which does not encrypt data in transit. This lack of encryption exposes sensitive information, such as login credentials and session tokens, to potential interception by attackers. Without HTTPS, any data exchanged between the client and server is vulnerable to eavesdropping and man-in-the-middle attacks.

#### Technical Detail
The site is served over HTTP, lacking the necessary HTTPS encryption. This configuration allows data transmitted between the client and server to be intercepted by attackers. The absence of HTTPS means that sensitive information, including authentication details, is transmitted in cleartext, making it susceptible to interception and misuse.

#### Evidence
No specific evidence provided.

#### Remediation
To address this vulnerability, configure the server to support HTTPS by obtaining a valid SSL/TLS certificate. Redirect all HTTP traffic to HTTPS to ensure that all data in transit is encrypted and secure. Implementing HTTPS will protect sensitive information from interception and unauthorized access. 

Standard guidance recommends implementing HTTPS to encrypt data in transit. Obtain and configure a valid SSL/TLS certificate for the server to ensure all data exchanged between the client and server is encrypted.
